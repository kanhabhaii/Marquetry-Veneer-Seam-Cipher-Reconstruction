# Dataset Description

## Overview

This dataset represents a synthetic marquetry workshop problem. Marquetry uses very thin sheets of decorative wood veneer. Several leaves may be cut together as a packet, then spread into a left-to-right strip so matching grain and seam marks line up. If the slips are shuffled after cutting, a technician must reconstruct the original order from edge evidence.

Each dataset row is one veneer packet. The public `piece_catalog` lists shuffled slip records. Real veneer slips have IDs beginning with `vslip_`; occasional filler records represent shop notes or guard strips and should be ignored. Each real slip has an encoded left seam (`L`) and encoded right seam (`R`). The left and right seams use different synthetic alphabets, so solvers must learn the translation from labeled training packets before reconstructing test packets.

The dataset is fully synthetic and generated by the included `generate_raw.py` script. It does not use public images, private workshop records, or scraped text. The preparation script hashes packet and slip identifiers, encodes raw seam signatures into public cipher tokens, creates deterministic train/test/private splits, and writes the public challenge files.

This challenge tests from-scratch structured reconstruction: parsing compact records, learning a seam-cipher relation, ignoring filler records, and producing a valid permutation. It uses ordered slip recovery rather than selected options or fixed labels.

## Prepared Challenge File Structure

These files are generated by `prepare.py`:

| File | Rows | Visibility | Description |
|---|---:|---|---|
| `public/train.csv` | 672 | Public | Labeled packets with `packet_id`, `piece_catalog`, and `leaf_order`. |
| `public/test.csv` | 288 | Public | Unlabeled packets with `packet_id` and `piece_catalog`. |
| `public/sample_submission.csv` | 288 | Public | Valid submission template with a simple fallback order. |
| `private/answers.csv` | 288 | Private | True test orders plus private metadata. |

## Raw Columns

| Column | Type | Description |
|---|---|---|
| `packet_id` | string | Raw packet identifier. Hashed by `prepare.py`. |
| `piece_catalog_raw` | string | Shuffled raw slip/filler records before public seam encoding. |
| `leaf_order_raw` | string | Pipe-delimited true order of the eight real raw leaf IDs. |
| `leaf_count` | integer | Number of real target leaves; always 8. |
| `difficulty` | string | Generator difficulty group: `plain`, `smudged`, or `busy`. |
| `start_left_raw` | string | Raw start-edge marker. Private after preparation. |
| `end_right_raw` | string | Raw end-edge marker. Private after preparation. |
| `boundary_indices` | string | Raw internal boundary indices used by the generator. Private after preparation. |

## Prepared Public Columns

| Column | Type | Files | Description |
|---|---|---|---|
| `packet_id` | string | train, test, sample_submission | Hashed packet identifier. |
| `piece_catalog` | string | train, test | Public shuffled catalog of real `vslip_` records plus possible filler records. Seam and descriptor fields are encoded by `prepare.py`. |
| `leaf_order` | string | train, sample_submission, answers | Pipe-delimited target order of the eight real `vslip_` IDs. |

## Notes

- The dataset is deterministic and synthetic.
- No external source data is used.
- The train/test split is deterministic and packet-level.
- Filler records are intentionally present in some packets to test careful parsing.
